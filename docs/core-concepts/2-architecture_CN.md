# 8 个 Lár 原语（“乐高积木”）

**`Lár` 不是一个繁重、复杂的框架。它是一个拥有 8 个核心“原语”的微型且强大的引擎。你可以组合这些“乐高积木”来构建任何智能体，从简单的聊天机器人到复杂的多智能体编排器。**

- **`GraphExecutor`（图执行器）**

  **它是什么**：“引擎”或“传送带”。

  **职责**：一个简单的生成器 (`generator`)，一次运行一个节点。它是驱动“智能”节点的“哑”循环。它也是“飞行数据记录仪”，为每一步 `yield`（产出）审计日志。

- **`GraphState`（图状态）**

  **它是什么**：“内存”或“剪贴板”。

  **职责**：一个简单的 Python 对象（字典包装器），会传递给每个节点。这是节点共享信息的方式。`LLMNode` 写入它，`ToolNode` 从中读取，`RouterNode` 利用它做出决策。

- **`LLMNode`（大模型节点）**

  **它是什么**：“大脑”或“思考者”。

  **职责**：该节点调用生成模型（如 Gemini）。它是进行推理、写作或分类的地方。我们的 `LLMNode` 具有弹性——它内置了指数退避机制，可在遇到 `429 速率限制错误` 时自动重试。它还会记录 Token 使用情况以便进行成本审计。

- **`ToolNode`（工具节点）**

  **它是什么**：“执行器”或“手臂”。

  **职责**：该节点运行任何简单的 Python 函数。这是你的智能体与世界交互的方式：运行代码、搜索数据库、调用 API，甚至是运行本地 `FAISS` 搜索。它拥有独立的 `next_node`（下一节点）和 `error_node`（错误节点）路径，使其非常稳健。

- **`RouterNode`（路由节点）**

  **它是什么**：“抉择”或“管理者”。

  **职责**：这是你的 `if/else` 语句。它运行一个简单的、确定性的 Python 函数（`decision_function`），该函数读取 `GraphState` 并返回一个字符串（例如 `"success"`）。然后，`RouterNode` 使用该字符串从其 `path_map`（路径映射）中选择下一个节点。

- **`BatchNode`（批处理节点）**

  **它是什么**：“加速器”或“并行器”。

  **职责**：在独立的线程中*并发*运行多个节点。它为每个线程创建状态副本（扇出 Fan-Out），并将结果合并回主状态（扇入 Fan-In）。这对于多智能体群（Swarms）至关重要。

- **`ReduceNode`（规约节点，v1.6+）**

  **它是什么**：“压缩器”或“总结者”。

  **职责**：在 Map-Reduce 流程中紧随 `BatchNode` 之后使用。它读取来自并行智能体的海量文档，使用 LLM 提取摘要，然后**显式地从状态中删除原始数据**，以防止上下文膨胀。

- **`HumanJuryNode`（人工陪审团节点，适配第 14 条）**

  **它是什么**：“硬件停止键”或“监督卫士”。

  **职责**：完全阻塞执行线程，直到人类通过 CLI（标准输入）或 API 显式提供“批准”或“拒绝”。它是唯一能够否决 LLM 的节点。为符合欧盟 AI 法案的要求而设计。

- **`"Utility" Nodes`（工具类节点）**

  **它们是什么**：简单的“助手”积木。

  `AddValueNode`：向状态中添加或复制数据。它是将 `{draft_answer}` 复制到 `{final_answer}` 的理想“终点”节点。

  `ClearErrorNode`：一个“清理”节点。其唯一职责是将 `last_error` 设置为 `None`，这对于构建自我修复循环至关重要。
