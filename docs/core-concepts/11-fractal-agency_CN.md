# 分形代理 (v1.5+)

虽然大多数智能体框架都在简单的循环或静态“聊天室”中运行，但 Lár 实现了**分形代理 (Fractal Agency)**：即图执行过程能够在运行时动态地递归生成相同的子图或全新的图拓扑结构。

这允许您在不扩大上下文窗口的情况下扩展复杂度。

## 核心理念

结合 v1.5+ 引入的 `DynamicNode` 和 `BatchNode`，智能体可以：

1. 分析一个高度复杂的问题。
2. 意识到该问题对于单一提示词或单一线性推理路径来说过于复杂。
3. 自动定义一个新的、专门的图（例如，分支成 5 个并行的研究员）。
4. 将该图作为一个孤立的子进程生成。
5. 等待子图完成，收集其统一的响应，然后恢复原始的顶级图运行。

### 为何这很重要：

*   **完美隔离**：“子智能体图”拥有自己的 `GraphState`。它零碎的草稿纸推理、迭代的工具调用以及肥大的检索文本**绝不会**污染主智能体的状态。它只返回精确、提炼后的答案。
*   **无限深度**：由于上下文是隔离的，子智能体可以继续生成子智能体，创建递归的“深度研究”树，而永远不会触发 Token 限制导致的崩溃。
*   **确定性架构**：尽管图在运行时无限扩展，但架构仍然严格遵循节点到节点的逻辑。Lár 强制 LLM 在严格的类型定义内输出这些拓扑结构。

## `fractal_polymath.py` 架构

这一能力的旗舰示例是“分形博学者 (Fractal Polymath)”。
您可以在 `examples/advanced/fractal_polymath.py` 中查看完整源代码。

在这种模式下，我们希望 AI 研究一个高度复杂的跨学科问题（例如，“小行星采矿的社会经济影响是什么？”）。

1.  **主编排器 (Master Orchestrator)** 定义问题。
2.  它使用 `DynamicNode` 递归地生成 $N$ 个专门的 `BatchNode` 子图（例如：经济学家、天体物理学家、律师）。
3.  这些智能体在各自的沙箱中完全并行运行。
4.  一旦所有子图执行完毕并终止，主编排器将它们孤立的输出合并为一个最终的权威摘要。

## 对分形实施预算控制

分形代理引入了死循环的危险可能性（智能体生成一个生成智能体的智能体，如此循环往复）。

Lár 通过强制执行 **Token 预算继承** 彻底消除了这一风险：

*   如果主图初始化时设置了 `token_budget=10,000`。
*   它消耗了 2,000 个 Token 来推理如何拆分工作。
*   此时它只剩下 8,000 个 Token。
*   当它动态生成 4 个并行的子智能体时，Lár 会将剩余的 8,000 个 Token 分配给它们，为每个子智能体设定 2,000 个 Token 的严格限制。
*   如果分形树中的*任何*节点违反了预算，整个树将优雅地停止并引发 `TokenBudgetExceededEvent`（Token 预算超出事件）。

这保证了递归 AI 架构内部的数学安全性。
