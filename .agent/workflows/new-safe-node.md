# `new-safe-node` 工作流

此工作流用于指导执行“创建一个新的 Lár 节点”任务。

1. **反向定义检查**：
   - 确定该节点的 **下一个节点** (`next_node`) 是否已定义。
   - 如果没有，先去写下一个节点的代码。
   
// turbo
2. **状态键定义**：
   - 列出该节点需要的 `input_keys` 并在代码中对应。
   - 列出该节点产出的 `output_key` 并在代码中声明。

3. **生成代码 (Reverse Order)**：
   - 确保节点定义在业务链条的下级之后。
   - 强制加入 Pydantic 类型提示：`(state: dict) -> dict`。
   - 强制加入异常处理，防止 `GraphExecutor` 在此步崩溃。

4. **安全过滤器集成 (Optional)**：
   - 如果是暴露给用户的节点，按照 `lar-security-auditor` 技能的要求，先加入一串正则过滤器节点。

5. **本地验证**：
   - 使用 `if __name__ == "__main__":` 写一段最小化的 `executor.run()` 用例来验证此单个节点的行为。
