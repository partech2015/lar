# `negentropy-loop-audit` 工作流

此工作流用于指导执行“评审动态生成的节点”或“执行系统负熵清理”的任务。

1. **变异代码审计**：
   - 查看 AI 或系统动态生成的 Python 代码。
   - 核查是否有任何非 JSON 序列化的复杂对象被直接丢入了 `state`（这会增加后续逻辑的熵，必须阻止）。

2. **状态崩塌检查 (The Collapse)**：
   - 计算该步执行前后的 `state` 键值库大小。
   - 如果键值对无意义地增多（例如：带入了大段网页原文但没有被后续节点清理），强制在图末尾增加一个 `ReduceNode` 或 `RemoveKeyNode`。

3. **并发预算对齐**：
   - 检查 `initial_state` 中的 `token_budget` 字段。
   - 如果当前节点的预估开销超过余量的 1/3，强制报警并建议降低模型等级。

// turbo
4. **沙盒 Dry-Run**：
   - 提议将此节点放入模拟器 (`SimulationExecutor`) 运行 3 次。
   - 对比平均时延和 Token 消耗。

5. **负熵决策**：
   - 如果新节点的熵值（Token/Latency/Complexity）小于原节点且准确率持平，建议执行“节点替换”操作。否则，保持原状。
