# 贡献 Lár

首先 —— 谢谢你！
Lár 的目标是成为 **代理推理界的 PyTorch**，社区的贡献至关重要。

## 如何贡献
### 1. Fork 仓库
### 2. 创建特性分支

```bash
git checkout -b feature/my-feature
```

### 3. 添加测试
Lár 强调可靠性和可审计性。
每个新特性**必须**包含测试。

### 4. 确保日志保持确定性
不要以破坏性的方式修改日志结构。

### 5. 提交 PR
包括：
- ⁠该特性的功能说明
- 为什么需要该特性
- ⁠变更前后的行为对比
- 示例用法

我们每周审查一次 PR。

## 代码风格
- ⁠Python 3.10+
- ⁠使用 Pydantic 定义模式
- ⁠必须使用类型提示 (Type hints)
- ⁠避免不必要的抽象
- ⁠遵循“玻璃盒子 (Glass Box)”哲学：一切都应该是可检查的

再次感谢 —— 你正在塑造代理的未来。
