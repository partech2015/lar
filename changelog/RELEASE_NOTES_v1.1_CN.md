# Lár v1.1 发布说明：“元认知更新”

我们很自豪地宣布 **Lár v1.1**，“元认知更新”正式发布。此版本引入了 4 级智能体（Level 4 Agency）能力，允许智能体在运行时修改自身的执行逻辑，同时保持严格的“玻盒”（Glass Box）合规性。

## 主要特性

### 1. 动态图（元认知）
智能体现现在执行过程中可以检查并修改自身的拓扑结构。
- **`DynamicNode`**：一个新的原语，允许节点返回一个*新的子图*或*拓扑修改*，而不仅仅是状态更新。
- **`TopologyValidator`**：一个确定性的验证引擎，确保任何自主组装的图都是有效、安全且可终止的，符合安全约束。
- **审计记录的自修改**：对图结构的每一次更改都会记录在“状态差异账本”（State-Diff Ledger）中，即使是自修改代码也能确保“玻盒”真实透明。

### 2. 新示例与“元认知”实验室
我们添加了 5 个展示动态图能力的高级示例：
- **`25_dynamic_depth.py`**：智能体会根据问题的复杂程度决定思考的“深度”。
- **`26_tool_inventor.py`**：智能体可以即时编写自己的 Python 工具来解决新颖的问题。
- **`27_self_healing.py`**：检测故障（如认证错误）的流水线，动态插入“修复”节点（如凭据轮转）并恢复执行。
- **`28_adaptive_deep_dive.py`**：智能体生成并行的子智能体，对其认为重要的课题进行研究。
- **`29_expert_summoner.py`**：智能体根据用户查询的主题动态实例化专门的“专家”节点。

### 3. 合规性与安全更新
- **兼容欧盟 AI 法案 (EU AI Act) 与 FDA**：更新了合规性文档以应对“自修改 AI”。通过使用 `TopologyValidator` 和不可变的“状态差异账本”，Lár v1.1 确保即使是动态行为也是完全可追溯和可复现的。
- **安全第一**：为在生产环境中部署动态智能体提供了新的文档警报和指南。

## 改进内容

- **示例重构**：所有示例已移动至 `lar/examples/` 并整理至清晰的分类中：
  - `basic/`：基础原语（线性、分支）。
  - `patterns/`：常见的智能体模式（RAG、ReAct、Map-Reduce）。
  - `compliance/`：安全与审计示例（人工陪审团、审计日志）。
  - `scale/`：高性能与分布式示例。
  - `metacognition/`：新的 v1.1 动态图示例。
- **网站更新**：
  - **红队挂钩**：在首页的合规（Compliance）部分添加了醒目的挂钩。
  - **文档改进**：修复了文档中警报/警告的渲染问题，并审计了所有链接。
  - **元认知章节**：为动态图添加了专门的章节。
- **版本升级**：框架版本更新至 `1.1.0`。

## 升级

要升级到 Lár v1.1，只需运行：
```bash
pip install lar-engine==1.1.0
```
